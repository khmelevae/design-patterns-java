# 🎮 Боевая система с паттерном Команда

## 📖 О проекте

Этот проект демонстрирует применение **паттерна Команда** (Command Pattern) на примере пошаговой боевой системы. В проекте представлены две реализации:

1. **Хорошая** — с использованием паттерна Команда (чистая архитектура)
2. **Плохая** — монолитный код без паттернов (для сравнения)

Проект создан в образовательных целях, чтобы показать преимущества использования поведенческих паттернов проектирования.

---

## 🎯 Цель демонстрации

Показать, как паттерн Команда помогает:

- ✅ Разделить ответственность между классами
- ✅ Сделать код легко расширяемым
- ✅ Реализовать отмену действий (Undo/Redo)
- ✅ Упростить тестирование
- ✅ Избавиться от гигантских `switch`/`if-else` конструкций

## 🎮 Игровая механика

### Основные правила

| Действие | Эффект | Урон | Особенности |
|----------|--------|------|-------------|
| **Удар** | Наносит урон противнику | 20 | Уменьшается вдвое при блоке |
| **Блок** | Защита и лечение | - | Восстанавливает 10 HP, снижает урон в 2 раза |
| **Спец. атака** | Мощная атака | 35 | Перезарядка 3 хода, уменьшается в 3 раза при блоке |
| **Отмена** | Возврат к предыдущему состоянию | - | Работает через Memento |

### Особенности реализации

#### 1. Паттерн Команда (Command)

Каждое действие игрока инкапсулируется в отдельный класс:

```java
// Пример команды удара
public class StrikeCommand extends BattleCommand {
    @Override
    public void execute() {
        state.saveState();  // Сохраняем состояние для Undo
        int damage = defender.takeDamage(attacker.getDamage());
        System.out.println(defender.getName() + " получает " + damage + " урона");
    }
}
```

**Преимущества:**
- ✅ Новые действия добавляются без изменения существующего кода
- ✅ Логика каждого действия изолирована
- ✅ Легко реализовать историю действий

#### 2. Паттерн Memento (для Undo)

`GameState` хранит снимки состояния персонажей:

```java
private static class Snapshot {
    int player1_health;
    int player2_health;
    boolean player1_isBlocking;
    boolean player2_isBlocking;
    int player1_cooldown;
    int player2_cooldown;
}
```

**Как работает:**
- Перед каждым ходом сохраняется состояние `saveState()`
- При отмене восстанавливается последнее сохранение `restoreState()`
- История хранится в стеке

## 🔍 Сравнение реализаций

### ❌ Плохая реализация (без паттернов)

```java
// Вся логика в одном методе
private void processTurn() {
    // 200+ строк кода с вложенными if-else
    if (choice == 1) {
        // Логика удара
        if (currentAttacker == 1) {
            if (p2Blocking) { /* ... */ }
            // ...
        }
    } else if (choice == 2) {
        // Логика блока
    } // и так далее...
}
```

#### Проблемы:
- 🔴 Громоздкий код — один метод содержит всю логику
- 🔴 Трудно расширять — нужно менять существующий код
- 🔴 Сложно тестировать — нельзя протестировать действие изолированно
- 🔴 Дублирование кода — похожие проверки повторяются
- 🔴 Хрупкость — изменение одного действия может сломать другие
- 🔴 История через строки — состояние сохраняется как строка, что ненадёжно

### ✅ Хорошая реализация (с паттернами)

```java
// Каждое действие — отдельный класс
Command command = new StrikeCommand(attacker, defender, gameState);
command.execute();
history.push(command);  // Сохраняем для Undo
```
#### Проблемы:
- 🟢 Чистый код — каждый класс отвечает за одно действие
- 🟢 Легко расширять — новый класс для нового действия
- 🟢 Просто тестировать — каждый тест проверяет одну команду
- 🟢 Нет дублирования — общая логика в абстрактном классе
- 🟢 Надёжный Undo — через Memento с сохранением объектов
- 🟢 Open/Closed Principle — классы открыты для расширения, закрыты для изменения

## 📚 Паттерны, использованные в проекте

### 1. Command (Команда)
Назначение: Превращает запросы в объекты, позволяя передавать их как аргументы, ставить в очередь, 
логировать и поддерживать отмену операций.

Где используется: Все действия игрока (удар, блок, спецатака) реализованы как команды.

### 2. Memento (Хранитель)
Назначение: Сохраняет и восстанавливает внутреннее состояние объекта без нарушения инкапсуляции.

Где используется: GameState сохраняет состояния персонажей для реализации Undo.

## 📝 Заключение
Паттерн Команда — это мощный инструмент для создания гибких и расширяемых систем. В этом проекте мы увидели, как он помогает:

- Организовать код в логические блоки
- Реализовать историю действий и отмену
- Упростить добавление новых функций
- Сделать код чище и поддерживаемее

Главный вывод: Использование паттернов проектирования — это не "усложнение ради усложнения", 
а способ сделать код более надёжным, понятным и готовым к изменениям.

- Всё в одном классе/методе

- Легко добавлять новые команды

- Код соответствует принципам SOLID
- Просто тестировать и поддерживать
